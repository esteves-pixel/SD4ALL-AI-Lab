# SD4ALL Data Lake

## Purpose
Central repository for all execution data generated by business processes.
This data feeds KPIs, dashboards, and AI agents.

## Structure

```
data/
├── README.md              ← You are here
├── raw/                   ← Raw data from source systems
│   ├── crm/               ← Student records, leads
│   ├── scheduling/        ← Class schedules, attendance
│   ├── finance/           ← Payments, invoices
│   └── marketing/         ← Campaign data, social metrics
├── processed/             ← Cleaned, transformed data
│   └── ...
├── kpis/                  ← KPI calculations and snapshots
│   └── ...
└── schemas/               ← Data models and validation
    └── ...
```

## Data Flow

```
Source Systems          Data Lake              Consumers
┌──────────┐    ETL    ┌──────────┐           ┌──────────┐
│ CRM      │ ────────→ │ raw/     │           │ KPI      │
│ Schedule │ ────────→ │ crm/     │──process─→│ Dashboard│
│ Payments │ ────────→ │ finance/ │           │          │
│ Website  │ ────────→ │ marketing│           │ AI Agents│
└──────────┘           └──────────┘           └──────────┘
```

## Principles
1. **All process data is captured** — if a process runs, it generates data
2. **Raw data is immutable** — never modify raw; create processed versions
3. **KPIs trace back to processes** — every metric links to a SIPOC process
4. **Agents consume, not own** — AI agents read data, humans own it
